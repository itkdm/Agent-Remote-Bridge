# 验收场景与发布门槛

本文档固化 `Agent Remote Bridge` 的三类端到端验收场景，以及每次直推 `main` 之前应执行的发布门槛。

## 发布门槛

本地基础门槛：

1. `python -m pytest`
2. 稳定工具 smoke
3. `agent_remote_bridge.main --help`
4. `agent_remote_bridge.main config-validate`

如果本地已经配置了可用远程主机，再追加：

5. `agent_remote_bridge.main preflight --host-id <HOST_ID> --timeout-sec 5`
6. `scripts/smoke_test.py --host-id <HOST_ID> --connect-only`

项目内置了统一入口：

```powershell
.\.venv\Scripts\python.exe .\scripts\release_gate.py --dry-run
.\.venv\Scripts\python.exe .\scripts\release_gate.py --host-id demo-server
```

说明：

- `--dry-run` 只打印标准检查，不执行
- 传入 `--host-id` 时，会追加远程预检和 `connect-only` smoke
- 如果远程链路本身存在 SSH banner、认证或网络问题，应按诊断结果排查，不把它误判成代码回归

## 场景 1：远程部署闭环

目标：

- 本地 Agent 能保留代码上下文
- 远程能完成基础部署动作
- 失败时能回到日志与状态检查

最小验收步骤：

1. `list_hosts`
2. `open_session(host_id="<HOST_ID>")`
3. `get_system_facts`
4. `exec_remote(command="pwd && whoami")`
5. 如需落地配置：`write_remote_file` 或 `append_remote_file`
6. `exec_remote(command="<deploy command>")`
7. 若失败：`check_service_status`、`tail_system_log`、`read_remote_file`

通过标准：

- 会话能正常打开和关闭
- 部署命令返回结构化结果
- 如果失败，能拿到统一错误类型和下一步建议

## 场景 2：502 排障闭环

目标：

- 能快速定位服务、系统日志和端口监听状态

最小验收步骤：

1. `open_session(host_id="<HOST_ID>")`
2. `check_service_status(service_name="<SERVICE>")`
3. `tail_system_log`
4. 如已开启实验工具：`check_port_listening(port=<PORT>)`、`inspect_processes(keyword="<KEYWORD>")`

通过标准：

- 服务状态结果可区分 `running`、`inactive`、`failed`、`not_found`
- 日志读取失败也必须返回结构化错误，不返回裸异常
- 审计记录里能看到失败阶段、错误类型和建议动作

## 场景 3：新机器初始化闭环

目标：

- 在新主机上完成最小环境识别和预检

最小验收步骤：

1. `config-validate`
2. `preflight --host-id <HOST_ID>`
3. `open_session`
4. `get_system_facts`
5. `exec_remote(command="uname -a && id && pwd")`

通过标准：

- `preflight` 能按 `config / dns / tcp / banner / auth` 给出阶段结果
- `get_system_facts` 能稳定返回基础环境信息
- 如果链路失败，必须明确失败阶段

## 维护节奏

每次功能单元完成后：

1. 跑发布门槛
2. 提交原子 commit
3. 打 `checkpoint/fNN-*` 标签
4. `git push origin main --follow-tags`
5. 若需回退，只用 `git revert <commit>`
